# Подробное ТЗ: UX и функциональное поведение расширения для сопоставления товаров Wildberries с Amazon ASIN

## 0. Назначение документа

Этот документ описывает финальное продуктовое и UX-ТЗ для расширения, которое помогает пользователю быстро сопоставлять товары Wildberries с товарами Amazon из собственной таблицы.

Документ описывает:

* пользовательские цели;
* основные сценарии;
* все элементы интерфейса;
* расположение кнопок на карточках Wildberries;
* поведение каждой кнопки;
* состояния карточек;
* tooltips, popovers, dropdown-меню, toast-уведомления;
* логику конфликтов;
* работу с группами;
* работу с активным ASIN;
* массовые действия;
* ожидаемые структуры данных на уровне продукта.

Документ **не описывает** конкретный стек, код, API, JS-архитектуру, детали авторизации, реализацию синхронизации, оптимизации DOM или конкретные библиотеки. Эти вопросы должны быть описаны отдельно.

Цель документа — исключить недосказанности в интерфейсе и сценариях, чтобы по нему можно было сделать реализацию или следующий технический документ без догадок.

---

# 1. Основная цель продукта

Пользователь имеет таблицу с перспективными товарами Amazon. Каждый Amazon-товар имеет ASIN и сопутствующие данные: название, изображение, бренд, комментарии, категорию, ключевые слова и рабочий статус.

Пользователь вручную ищет на Wildberries товары, которые:

* совпадают с Amazon-товаром;
* похожи на Amazon-товар;
* являются конкурентами;
* могут быть полезны для дальнейшего анализа;
* не подходят, но должны быть зафиксированы как просмотренные или отклонённые.

Расширение должно заменить ручной процесс копирования ссылок WB в таблицу. Вместо этого пользователь должен работать прямо на странице Wildberries и связывать WB-карточки с Amazon ASIN одним или несколькими быстрыми действиями.

Главный принцип:

> Пользователь выбирает активный Amazon ASIN, открывает Wildberries, листает карточки и одним кликом на карточке WB добавляет найденный WB-товар к выбранному ASIN.

# 2. Основные принципы интерфейса

## 2.1. Минимальное вмешательство в страницу Wildberries

Расширение не должно заметно перестраивать страницу Wildberries.

Запрещённые или нежелательные подходы:

* постоянная боковая панель, которая сдвигает контент WB;
* крупные блоки между фото, ценой и названием карточки;
* текстовые плашки прямо на фото товара;
* элементы, которые увеличивают высоту карточки;
* элементы, которые ломают сетку карточек;
* элементы, которые перекрывают важные штатные элементы WB без необходимости.

Допустимые подходы:

* компактный ряд иконок внутри или рядом с областью карточки;
* небольшие overlay-иконки поверх фото или у нижнего края фото;
* dropdown-меню по кнопке `⋯`;
* tooltips при наведении;
* popover для выбора ASIN или группы;
* временная плавающая панель управления, которая открывается поверх страницы и не сдвигает контент.

## 2.2. Основная логика управления

На карточке WB должно быть минимальное количество видимых элементов:

```text
[статусные иконки]      [быстрые действия]
👁  A  ≡  ×  ?           A+  ⋯
```

Не все иконки должны быть видны всегда. Видимость зависит от состояния товара.

Главные элементы:

* `A+` — быстро привязать WB-товар к активному ASIN.
* `⋯` — открыть дополнительные действия по конкретной карточке.
* `👁` — товар был просмотрен пользователем или с ним было взаимодействие.
* `A` — товар уже связан с одним или несколькими ASIN.
* `≡` — товар находится в одной или нескольких пользовательских группах.
* `×` — товар отклонён.
* `?` или `⏳` — товар отложен / требует проверки.

## 2.3. Основной рабочий контекст

Главный рабочий контекст — **активный Amazon ASIN**.

Пользователь выбирает активный ASIN в плавающей панели управления. После этого кнопка `A+` на любой карточке WB означает:

> Добавить этот WB-товар к текущему активному ASIN.

Если активный ASIN не выбран, кнопка `A+` не должна выполнять привязку. Она должна показать tooltip или popover с объяснением, что сначала нужно выбрать ASIN.

# 3. Термины и сущности

## 3.1. WB-товар

WB-товар — товар Wildberries, найденный на странице WB.

Ключ WB-товара: артикул / SKU / идентификатор товара, извлечённый из ссылки на карточку WB.

Для каждого WB-товара система должна хранить или отображать:

* WB SKU;
* ссылку на WB;
* изображение, если доступно;
* название, если доступно;
* цену, если доступна;
* бренд, если доступен;
* факт просмотра пользователем (`seen`);
* связи с ASIN;
* принадлежность к группам;
* статус `rejected`, если товар отклонён;
* статус `deferred`, если товар отложен;
* дату первого взаимодействия;
* дату последнего взаимодействия;
* историю событий.

## 3.2. Amazon-товар

Amazon-товар — товар из пользовательской таблицы перспективных товаров Amazon.

Основной ключ: ASIN.

Для каждого Amazon-товара желательно иметь:

* ASIN;
* название;
* изображение;
* бренд;
* категория;
* комментарий пользователя;
* ключевые слова;
* статус в рабочей очереди;
* приоритет;
* дата последней проверки;
* итог проверки;
* количество связанных WB-товаров.

## 3.3. Активный ASIN

Активный ASIN — Amazon-товар, выбранный пользователем в плавающей панели как текущая цель поиска.

Все быстрые действия `A+` на карточках WB относятся именно к активному ASIN.

Если пользователь выбирает другой активный ASIN, визуальное состояние карточек WB должно обновиться относительно нового контекста:

* карточки, уже связанные с новым активным ASIN, показывают состояние `linked to active ASIN`;
* карточки, связанные с другим ASIN, показывают состояние `linked to other ASIN` или `conflict`, если пользователь пытается привязать их к текущему ASIN;
* карточки без связей показывают обычное состояние.

## 3.4. Связь WB ↔ ASIN

Связь означает, что конкретный WB-товар сопоставлен с конкретным Amazon ASIN.

Связь должна иметь тип. Минимальный набор типов:

1. `candidate` — кандидат, требует проверки.
2. `exact_match` — тот же товар или максимально точное совпадение.
3. `similar` — похожий товар.
4. `competitor` — конкурент или аналог.
5. `wrong_size` — похожий товар, но отличается размером/объёмом/комплектацией.
6. `wrong_product` — оказался не тем товаром, но связь сохранена как отрицательное наблюдение, если это нужно пользователю.

Для быстрого действия `A+` должен быть задан тип связи по умолчанию. Рекомендуемый вариант по умолчанию: `candidate`. В настройках панели можно позволить пользователю выбрать тип по умолчанию.

## 3.5. Группа

Группа — пользовательская подборка WB-товаров.

Группа не является заменой ASIN-связи. Товар может быть:

* только в группе;
* только связан с ASIN;
* одновременно связан с ASIN и состоять в одной или нескольких группах;
* отклонён и всё равно находиться в группе, если пользователь сознательно так сделал.

Группа должна иметь:

* название;
* иконку;
* комментарий;
* дату создания;
* дату последнего изменения;
* счётчик товаров;
* опциональный тип группы.

Примеры групп:

* `Проверить позже`;
* `Интересные конкуренты`;
* `Похожие, но не точные`;
* `Хорошие фото`;
* `Другой размер`;
* `Другой бренд`;
* `Потенциально опасно`.

## 3.6. Seen

`Seen` означает, что пользователь реально обратил внимание на товар или взаимодействовал с ним.

`Seen` не означает, что товар обработан, принят, отклонён или связан. Это только факт пользовательского внимания.

Товар получает `seen`, если произошло хотя бы одно из событий:

* пользователь навёл мышь на карточку дольше заданного порога `N` секунд;
* пользователь открыл меню `⋯` на карточке;
* пользователь нажал любую кнопку на карточке;
* пользователь нажал `A+`;
* пользователь добавил товар к ASIN;
* пользователь добавил товар в группу;
* пользователь отклонил товар;
* пользователь отложил товар;
* пользователь скопировал ссылку WB;
* пользователь открыл историю карточки;
* пользователь выделил карточку чекбоксом для массового действия.

Рекомендуемое значение `N`: 1.2–1.5 секунды. В ТЗ это можно описать как настраиваемый параметр, но в интерфейсе мокапа использовать 1.2 секунды.

# 4. Общий layout на странице WB

## 4.1. Сетка карточек

Расширение не должно менять основную сетку WB.

Требования:

* карточки должны оставаться в штатной сетке WB;
* высота карточек не должна искусственно увеличиваться из-за элементов расширения;
* элементы расширения не должны обрезаться родительскими контейнерами;
* dropdown, tooltip и popover должны отображаться поверх страницы;
* элементы расширения должны иметь достаточный приоритет слоя, чтобы быть видимыми;
* элементы расширения не должны перекрывать цену, название, рейтинг и штатные кнопки WB.

## 4.2. Зоны карточки

Карточка WB логически делится на зоны:

```text
┌─────────────────────────────┐
│ Фото товара                 │
│                             │
│ Нижний край фото: иконки    │
├─────────────────────────────┤
│ Цена                        │
│ Бренд / название            │
│ Рейтинг / доставка / прочее │
└─────────────────────────────┘
```

Элементы расширения должны размещаться в зоне фото, ближе к нижнему краю изображения, либо в компактной overlay-строке внутри карточки. Они не должны добавлять новую строку под фото, если это ломает высоту сетки.

## 4.3. Рекомендуемая схема размещения элементов на карточке

```text
┌──────────────────────────────┐
│ фото товара              ♡   │
│                              │
│                              │
│ 👁  A  ≡  ×  ?      A+  ⋯    │
└──────────────────────────────┘
Цена
Бренд
Название
Рейтинг
```

Левая часть нижнего ряда — статусные иконки.

Правая часть нижнего ряда — быстрые действия.

Если места мало:

```text
┌──────────────────────────────┐
│ фото товара              ♡   │
│                              │
│ 👁 A ≡ × ?              A+ ⋯  │
└──────────────────────────────┘
```

Если карточка узкая, допускается скрывать часть неактуальных статусных иконок, но нельзя скрывать `A+` и `⋯`, если карточка является релевантной для обработки.

# 5. Кнопки и иконки на карточке товара

## 5.1. Общие правила для всех иконок

Каждая иконка должна иметь:

* понятный визуальный вид;
* tooltip при hover;
* доступное состояние `disabled`, если действие невозможно;
* состояние hover;
* состояние active/click;
* не менее одной визуальной реакции на действие;
* запись события в историю, если действие меняет данные.

Иконки должны быть компактными. Рекомендуемый визуальный размер: 20–28 px.

Все иконки должны быть кликабельны только тогда, когда они действительно выполняют действие. Если иконка только информационная, это должно быть ясно: курсор, tooltip и отсутствие ложного affordance.

## 5.2. Иконка `○` — новый / нет данных

### Значение

Товар ещё не имеет сохранённого контекста в базе или не был обработан пользователем.

### Видимость

Иконка `○` показывается, если у товара нет следующих признаков:

* `seen`;
* ASIN-связей;
* групп;
* `rejected`;
* `deferred`.

Если на карточке есть любой более конкретный статус, `○` не обязателен и может быть скрыт.

### Tooltip

```text
Новый товар
Нет сохранённого контекста
```

Если система уже знает SKU, но пользователь не взаимодействовал:

```text
Товар найден на странице
Пользователь ещё не взаимодействовал с ним
```

### Поведение по клику

По умолчанию иконка `○` не должна быть основной кнопкой. Она может быть только информационной.

Если иконка кликабельна, клик должен означать `Отметить как seen`. В этом случае tooltip должен явно говорить:

```text
Отметить как просмотренный
```

## 5.3. Иконка `👁` — seen

### Значение

Пользователь уже взаимодействовал с товаром или удерживал hover на карточке дольше N секунд.

### Видимость

Показывается, если у товара есть признак `seen = true`.

### Tooltip без дополнительных действий

```text
Уже видели
Первое взаимодействие: {date_time}
Последнее взаимодействие: {date_time}
```

### Tooltip, если товар seen, но не обработан

```text
Уже видели
Связей с ASIN нет
В группах не состоит
Решение ещё не принято
```

### Tooltip, если товар seen и обработан

```text
Уже видели
Есть сохранённый контекст
Наведи на A, ≡, × или ? для деталей
```

### Поведение

* Hover показывает tooltip.
* Клик может открыть краткий контекст товара, если это предусмотрено.
* Если клик не используется, иконка должна быть только информационной.

## 5.4. Иконка `A` — есть ASIN-связь

### Значение

WB-товар связан хотя бы с одним Amazon ASIN.

### Видимость

Показывается, если существует хотя бы одна связь WB ↔ ASIN.

### Варианты состояния

#### 5.4.1. Связан с активным ASIN

Иконка: `A`

Tooltip:

```text
Привязан к текущему ASIN
{ASIN}
{Amazon title}
Тип связи: {link_type}
Дата связи: {date_time}
```

Если есть несколько связей, среди которых есть активный ASIN:

```text
Есть связь с текущим ASIN
{ASIN} — {Amazon title}
Дополнительно связан ещё с {count} ASIN
```

#### 5.4.2. Связан с другим ASIN

Иконка: `A`

Tooltip:

```text
Привязан к другому ASIN
{ASIN}
{Amazon title}
Тип связи: {link_type}
```

Если активный ASIN выбран, tooltip должен добавить:

```text
Текущий активный ASIN: {active_asin}
```

#### 5.4.3. Потенциальный конфликт

Иконка: `A!` или `A` с предупреждающим стилем.

Показывается, если:

* карточка уже связана с одним ASIN;
* активный ASIN выбран;
* активный ASIN отличается от уже связанного;
* пользователь пытается выполнить действие `A+` или массовую привязку к активному ASIN.

Tooltip:

```text
Возможный конфликт
Товар уже связан с {existing_asin}
Активный ASIN: {active_asin}
```

### Поведение по клику

Клик по `A` должен открыть popover с деталями ASIN-связей.

Popover:

```text
Связи с ASIN

1. {ASIN}
   {Amazon title}
   Тип: {link_type}
   Дата: {date_time}

[Открыть Amazon-контекст]
[Изменить тип связи]
[Удалить связь]
```

Если связей несколько, они отображаются списком.

Удаление связи должно требовать подтверждения.

## 5.5. Иконка `≡` — товар в группе

### Значение

WB-товар добавлен в одну или несколько пользовательских групп.

### Видимость

Показывается, если товар находится хотя бы в одной группе.

### Tooltip для одной группы

```text
В группе
{group_icon} {group_name}
{group_comment}
```

### Tooltip для нескольких групп

```text
В группах:
{group_icon} {group_name_1}
{group_icon} {group_name_2}
{group_icon} {group_name_3}
```

Если групп больше трёх:

```text
И ещё {count} групп
```

### Поведение по клику

Клик по `≡` открывает popover:

```text
Группы товара

✓ {group_icon} {group_name_1}
✓ {group_icon} {group_name_2}

[Добавить в группу]
[Убрать из группы]
[Создать новую группу]
```

При выборе `Убрать из группы` должен открываться список групп с чекбоксами.

## 5.6. Иконка `×` — rejected

### Значение

WB-товар отклонён пользователем.

### Видимость

Показывается, если товар имеет статус `rejected`.

### Tooltip

```text
Отклонён
Причина: {reason}
Дата: {date_time}
```

Если причина не указана:

```text
Отклонён
Причина не указана
```

### Поведение по клику

Клик по `×` открывает popover:

```text
Товар отклонён
Причина: {reason}

[Снять отклонение]
[Изменить причину]
[Показать историю]
```

Снятие отклонения должно требовать подтверждения или показывать undo-toast.

## 5.7. Иконка `?` или `⏳` — deferred / проверить позже

### Значение

Товар отложен для дальнейшей проверки.

### Видимость

Показывается, если товар имеет статус `deferred`.

### Tooltip

```text
Отложен / требует проверки
Причина: {reason}
Дата: {date_time}
```

Если товар отложен до определённой даты:

```text
Проверить после: {date}
```

### Поведение по клику

Клик открывает popover:

```text
Отложенный товар
Причина: {reason}

[Снять отложенный статус]
[Изменить причину]
[Добавить в группу Проверить позже]
```

## 5.8. Кнопка `A+` — быстро добавить к активному ASIN

### Значение

Главная кнопка карточки. Выполняет быстрое сопоставление WB-товара с активным ASIN.

### Расположение

Правая часть нижнего overlay-ряда карточки, перед кнопкой `⋯`.

Пример:

```text
👁  A  ≡              A+  ⋯
```

### Видимость

Показывается на всех релевантных WB-карточках.

### Состояния

#### Активный ASIN выбран, товар не связан

Кнопка активна.

Tooltip:

```text
Добавить к активному ASIN
{active_asin}
{Amazon title}
```

Клик:

1. товар получает `seen = true`;
2. создаётся связь WB ↔ active ASIN;
3. тип связи ставится по умолчанию;
4. на карточке появляется или обновляется иконка `A`;
5. показывается toast с возможностью отмены.

Toast:

```text
Товар привязан к {active_asin}
[Отменить]
```

#### Активный ASIN выбран, товар уже связан с этим ASIN

Кнопка активна, но не должна создавать дубль.

Tooltip:

```text
Уже привязан к текущему ASIN
{active_asin}
```

Клик:

Показать toast:

```text
Связь уже существует
```

Новые данные не создаются.

#### Активный ASIN выбран, товар связан с другим ASIN

Клик не должен молча заменять связь.

При клике открывается conflict-popover:

```text
У товара уже есть связь

Текущая связь:
{existing_asin}
{existing_amazon_title}

Вы хотите добавить связь с:
{active_asin}
{active_amazon_title}

[Оставить как есть]
[Добавить вторую связь]
[Заменить связь]
[Отмена]
```

Поведение кнопок:

* `Оставить как есть` — закрыть popover, ничего не менять.
* `Добавить вторую связь` — создать дополнительную связь, старую сохранить.
* `Заменить связь` — удалить или деактивировать старую связь и создать новую.
* `Отмена` — закрыть popover.

После `Добавить вторую связь` toast:

```text
Добавлена дополнительная связь с {active_asin}
[Отменить]
```

После `Заменить связь` toast:

```text
Связь заменена на {active_asin}
[Отменить]
```

#### Активный ASIN не выбран

Кнопка визуально доступна, но действие не выполняется напрямую.

Tooltip:

```text
Сначала выберите активный ASIN в панели
```

Клик открывает плавающую панель выбора ASIN или popover:

```text
Активный ASIN не выбран

[Выбрать ASIN]
[Добавить к ASIN через поиск]
```

`Выбрать ASIN` открывает панель управления.

`Добавить к ASIN через поиск` открывает карточечный ASIN-search popover для конкретного WB-товара.

#### Товар rejected

Если товар отклонён, `A+` не должен молча привязывать его без подтверждения.

Клик открывает popover:

```text
Товар отмечен как отклонённый

Причина: {reason}

[Снять отклонение и привязать]
[Привязать, оставив отклонённым]
[Отмена]
```

Рекомендуемое поведение по умолчанию: требовать явного выбора.

## 5.9. Кнопка `⋯` — меню действий карточки

# 6. Подробное поведение пунктов меню `⋯`

## 6.1. `Привязать к активному ASIN`

Дублирует действие `A+`, но находится в меню.

Используется, если пользователь привык работать через меню или если `A+` визуально не используется.

Поведение полностью совпадает с кнопкой `A+`.

## 6.2. `Добавить к ASIN...`

### Назначение

Позволяет привязать WB-товар не к активному ASIN, а к любому выбранному ASIN через поиск.

Нужно для нестандартных рекомендаций WB, когда пользователь видит, что найденный товар лучше подходит не к текущему активному ASIN, а к другому Amazon-товару.

### Открываемый интерфейс

После клика открывается ASIN-search popover, привязанный к карточке.

Popover не должен сдвигать страницу. Он должен отображаться поверх страницы.

Заголовок:

```text
Добавить товар к ASIN
```

Внутри:

```text
[строка поиска]

Результаты:
[карточка ASIN 1]
[карточка ASIN 2]
[карточка ASIN 3]
```

### Строка поиска

Placeholder:

```text
ASIN, название, бренд, комментарий...
```

Поиск должен работать по всем доступным полям Amazon-товара:

* ASIN;
* title;
* brand;
* comment;
* category;
* keywords;
* workflow status;
* любым дополнительным полям, которые будут в таблице.

Пользователь не должен выбирать режим поиска. Система сама определяет, что введено:

* если строка похожа на ASIN — приоритет ASIN;
* если строка похожа на бренд — приоритет бренда;
* если строка содержит обычные слова — поиск по названию, комментариям и ключевым словам.

### Если строка поиска пустая

Показывается активная выборка:

```text
Недавние и активные ASIN
```

В выборку должны попадать:

1. активный ASIN, если он выбран;
2. недавно использованные ASIN;
3. ASIN с наибольшим числом добавлений за последние 30 дней;
4. ASIN из текущей рабочей очереди;
5. ASIN со статусом `в работе`;
6. последние доступные ASIN, если нет данных за 30 дней.

### Карточка результата ASIN

Каждый результат должен быть визуальной плиткой:

```text
[фото]
{ASIN}
{title}
Бренд: {brand}
Комментарий: {comment}
Статус: {workflow_status}

[Добавить]
```

Минимальный состав:

* фото;
* ASIN;
* title;
* бренд;
* кнопка `Добавить`.

Если данных нет, поле скрывается или показывается как `—`, но не должно ломать карточку.

### Выбор результата

Клик по карточке результата или кнопке `Добавить`:

1. товар WB получает `seen = true`;
2. создаётся связь WB ↔ выбранный ASIN;
3. тип связи ставится по умолчанию или выбирается пользователем, если включён режим подтверждения;
4. popover закрывается;
5. карточка WB обновляет иконку `A`;
6. показывается toast.

Toast:

```text
Товар привязан к {selected_asin}
[Отменить]
```

### Если есть конфликт

Если WB-товар уже связан с другим ASIN, перед созданием новой связи показывается conflict-popover:

```text
У товара уже есть связь

Существующая связь:
{existing_asin}
{existing_title}

Новая связь:
{selected_asin}
{selected_title}

[Оставить как есть]
[Добавить вторую связь]
[Заменить связь]
[Отмена]
```

### Если выбран тот же ASIN

Если связь уже существует:

```text
Связь уже существует
```

Дубликат не создаётся.

### Назначение

Позволяет добавить WB-товар в одну или несколько пользовательских групп.

Поиск должен работать по:

* комментарию;
* иконке/типу группы;
* частоте использования;
* недавним добавлениям.

Показываются:

```text
[Отмена]
```

После создания:

1. создаётся группа;
2. товар добавляется в эту группу;
3. на карточке появляется `≡`;

# 7. Плавающая панель управления

## 7.1. Назначение панели

Панель нужна для управления активным ASIN, очередью Amazon-товаров, статистикой страницы и массовыми действиями.

Панель не должна быть постоянной боковой колонкой. Она открывается и закрывается поверх страницы.

## 7.2. Кнопка открытия панели

### Расположение

Фиксированно в правом верхнем углу страницы.

Пример:

```text
WB ↔ A
```

или компактная кнопка с иконкой.

### Требования

* кнопка всегда видима при скролле;
* кнопка не должна перекрывать важные элементы WB, насколько это возможно;
* кнопка должна иметь высокий z-index;
* кнопка должна открывать и закрывать панель;
* состояние панели должно быть очевидным.

### Tooltip

```text
Панель сопоставления WB ↔ Amazon
```

## 7.3. Панель в открытом состоянии

Панель отображается как прямоугольник поверх страницы, справа сверху или справа сбоку.

Она не сдвигает страницу.

Рекомендуемая структура:

```text
┌────────────────────────────────────┐
│ WB ↔ Amazon                    [×] │
├────────────────────────────────────┤
│ Активный ASIN                      │
│ [поиск по ASIN / названию / бренду]│
│                                    │
│ [фото] B0C7G00SE12                 │
│       Название товара              │
│       Бренд                        │
│       Комментарий                  │
│                                    │
│ [← Предыдущий] [Следующий →]       │
├────────────────────────────────────┤
│ Режим A+                           │
│ ● сразу привязывать                │
│ ○ спрашивать подтверждение         │
│                                    │
│ Тип связи по умолчанию             │
│ [candidate ▼]                      │
├────────────────────────────────────┤
│ Статистика страницы                │
│ Всего карточек: X                  │
│ Уникальных WB SKU: X               │
│ Новых: X                           │
│ Seen: X                            │
│ Связаны с ASIN: X                  │
│ В группах: X                       │
│ Отклонены: X                       │
│ Отложены: X                        │
├────────────────────────────────────┤
│ Массовые действия                  │
│ Выбрано: N                         │
│ [К активному ASIN]                 │
│ [К выбранному ASIN...]             │
│ [В группу...]                      │
│ [Отклонить]                        │
│ [Снять выбор]                      │
└────────────────────────────────────┘
```

Если нет выбранных карточек, блок массовых действий должен быть скрыт или показывать:

```text
Массовые действия появятся после выбора карточек
```

## 7.4. Поиск активного ASIN в панели

### Назначение

Пользователь выбирает Amazon-товар, который станет активным ASIN.

### Поиск

Строка поиска работает по:

* ASIN;
* названию;
* бренду;
* комментариям;
* категории;
* ключевым словам;
* статусу в очереди.

Placeholder:

```text
Найти ASIN, товар, бренд...
```

### Пустой поиск

Если строка пустая, показываются:

1. текущий активный ASIN;
2. следующий товар в очереди;
3. товары со статусом `в работе`;
4. недавно использованные ASIN;
5. ASIN с наибольшими добавлениями за последние 30 дней.

### Выбор ASIN

Клик по результату делает его активным.

После выбора:

* карточки на странице обновляют относительное состояние;
* `A+` начинает привязывать к новому ASIN;
* панель показывает данные выбранного ASIN;
* показывается toast.

Toast:

```text
Активный ASIN выбран: {asin}
```

## 7.5. Очередь Amazon-товаров

Панель должна поддерживать переходы:

```text
[← Предыдущий] [Следующий →]
```

Очередь может быть основана на таблице Amazon-товаров и их workflow status.

Рекомендуемые статусы Amazon-товара:

* `not_started` — не начат;
* `in_progress` — в работе;
* `matched` — найден точный WB-аналог;
* `candidates_found` — найдены кандидаты;
* `competitors_found` — найдены конкуренты;
* `not_found` — проверено, не найдено;
* `deferred` — отложено;
* `done` — завершено.

В панели можно показывать и изменять статус активного ASIN.

## 7.6. Режим кнопки `A+`

Панель должна иметь настройку режима `A+`:

1. `Сразу привязывать`.
2. `Спрашивать подтверждение`.

### Сразу привязывать

Обычный режим для максимальной скорости.

Клик `A+` сразу создаёт связь, если нет конфликта.

Если есть конфликт, подтверждение всё равно обязательно.

### Спрашивать подтверждение

Клик `A+` открывает popover:

```text
Привязать товар к ASIN?

WB: {wb_title}
ASIN: {active_asin}
{amazon_title}
Тип связи: [candidate ▼]

[Привязать]
[Отмена]
```

## 7.7. Тип связи по умолчанию

В панели должен быть выбор:

```text
Тип связи по умолчанию
[candidate ▼]
```

Варианты:

* `candidate`;
* `exact_match`;
* `similar`;
* `competitor`;
* `wrong_size`;
* `wrong_product`.

Выбранный тип применяется к быстрым связям через `A+`.

# 8. Массовые действия

## 8.1. Выбор карточек

Каждая карточка может иметь компактный чекбокс выбора.

Расположение чекбокса:

* верхний левый угол фото;
* либо рядом с рядом иконок;
* чекбокс не должен перекрывать критичные WB-элементы.

Если чекбокс выбран:

* карточка получает лёгкое визуальное выделение;
* товар получает `seen = true`;
* в панели появляется блок массовых действий.

## 8.2. Блок массовых действий в панели

Если выбрано N товаров:

```text
Выбрано: {N}

[К активному ASIN]
[К выбранному ASIN...]
[В группу...]
[Отклонить]
[Отложить]
[Снять выбор]
```

## 8.3. `К активному ASIN`

### Если активный ASIN выбран

Клик запускает проверку выбранных товаров.

Если конфликтов нет:

```text
Привязать {N} товаров к {active_asin}?

Тип связи: [candidate ▼]

[Привязать]
[Отмена]
```

Если режим быстрых действий допускает без подтверждения, подтверждение всё равно рекомендуется для массового действия.

### Если активный ASIN не выбран

Показать сообщение:

```text
Активный ASIN не выбран

[Выбрать ASIN]
[Отмена]
```

## 8.4. `К выбранному ASIN...`

### Назначение

Массово привязать выбранные WB-товары к ASIN, выбранному через поиск, а не обязательно к активному ASIN.

### Интерфейс

Открывается ASIN-search popover в панели.

Заголовок:

```text
Привязать выбранные товары к ASIN
```

Подзаголовок:

```text
Выбрано товаров: {N}
```

Поиск работает так же, как в карточечном меню `Добавить к ASIN...`.

После выбора ASIN:

```text
Привязать {N} товаров к {selected_asin}?

Тип связи: [candidate ▼]

[Привязать]
[Отмена]
```

## 8.5. Конфликты при массовой привязке

Перед выполнением нужно разделить выбранные товары на категории:

1. нет ASIN-связей;
2. уже связаны с выбранным ASIN;
3. связаны с другим ASIN;
4. rejected;
5. deferred;
6. уже имеют несколько связей.

Если есть конфликты, показать summary:

```text
Перед привязкой найдены конфликты

Выбрано товаров: {N}
Без конфликтов: {count}
Уже связаны с этим ASIN: {count}
Связаны с другим ASIN: {count}
Отклонены: {count}
Отложены: {count}

Что сделать?

[Привязать только без конфликтов]
[Добавить вторую связь конфликтным]
[Заменить связи конфликтным]
[Показать список]
[Отмена]
```

По умолчанию безопасный вариант:

```text
Привязать только без конфликтов
```

Нельзя молча заменять существующие связи.

## 8.6. `В группу...`

Открывает group-search popover.

После выбора группы:

```text
Добавить {N} товаров в группу «{group_name}»?

[Добавить]
[Отмена]
```

Если часть товаров уже в группе:

```text
Некоторые товары уже есть в группе

Новых добавлений: {new_count}
Уже были в группе: {existing_count}

[Добавить только новые]
[Отмена]
```

## 8.7. `Отклонить`

Открывает массовый reject-popover:

```text
Отклонить выбранные товары

Выбрано: {N}
Причина:
[выбрать или ввести]

[Отклонить]
[Отмена]
```

Если среди выбранных есть ASIN-связи:

```text
Среди выбранных есть товары со связями ASIN: {count}

[Отклонить и оставить связи]
[Отклонить и удалить связи]
[Отмена]
```

По умолчанию — оставить связи.

## 8.8. `Отложить`

Открывает массовый deferred-popover:

```text
Отложить выбранные товары

Выбрано: {N}
Причина:
[выбрать или ввести]

[Отложить]
[Отмена]
```

## 8.9. `Снять выбор`

Снимает выделение со всех выбранных карточек.

Toast не обязателен. Можно показать кратко:

```text
Выбор снят
```

# 9. Состояния карточки и матрица поведения

## 9.1. Состояние `new`

### Условия

* товар найден на странице;
* пользователь с ним не взаимодействовал;
* нет сохранённых связей;
* нет групп;
* нет rejected/deferred.

### Иконки

```text
○              A+ ⋯
```

### Tooltip `○`

```text
Новый товар
Нет сохранённого контекста
```

### `A+`

Если активный ASIN выбран — привязать к нему.

Если не выбран — предложить выбрать ASIN.

### `⋯`

Полное меню действий.

## 9.2. Состояние `seen only`

### Условия

* `seen = true`;
* нет ASIN-связей;
* нет групп;
* не rejected;
* не deferred.

### Иконки

```text
👁              A+ ⋯
```

### Tooltip `👁`

```text
Уже видели
Связей с ASIN нет
В группах не состоит
```

### `A+`

Привязать к активному ASIN.

## 9.3. Состояние `linked to active ASIN`

### Условия

* есть связь с активным ASIN.

### Иконки

```text
👁 A            A+ ⋯
```

### Tooltip `A`

```text
Привязан к текущему ASIN
{active_asin}
{amazon_title}
Тип связи: {link_type}
```

### `A+`

Не создаёт дубль.

Toast:

```text
Связь уже существует
```

## 9.4. Состояние `linked to other ASIN`

### Условия

* активный ASIN выбран;
* товар связан с ASIN, отличным от активного.

### Иконки

```text
👁 A!           A+ ⋯
```

или

```text
👁 A            A+ ⋯
```

при этом `A` визуально предупреждает.

### Tooltip `A`

```text
Привязан к другому ASIN
{existing_asin}
{existing_title}

Активный ASIN:
{active_asin}
{active_title}
```

### `A+`

Открывает conflict-popover.

## 9.5. Состояние `in group`

### Условия

* товар находится в одной или нескольких группах;
* может не иметь ASIN-связей.

### Иконки

```text
👁 ≡            A+ ⋯
```

### Tooltip `≡`

Показывает группы.

### `A+`

Привязка к ASIN разрешена без конфликта, если нет ASIN-связей.

## 9.6. Состояние `rejected`

### Условия

* товар отклонён.

### Иконки

```text
👁 ×            A+ ⋯
```

### Tooltip `×`

```text
Отклонён
Причина: {reason}
```

### `A+`

Не должен молча привязывать. Требует подтверждения:

```text
Товар отклонён

[Снять отклонение и привязать]
[Привязать, оставив отклонённым]
[Отмена]
```

## 9.7. Состояние `deferred`

### Условия

* товар отложен для проверки.

### Иконки

```text
👁 ?            A+ ⋯
```

### `A+`

Разрешён. После привязки deferred может сохраниться, если пользователь не снял статус.

При клике можно показывать небольшой warning, если включён режим подтверждения:

```text
Товар отложен для проверки
Привязать всё равно?
```

## 9.8. Состояние `linked + in group`

### Условия

* есть ASIN-связь;
* есть группа.

### Иконки

```text
👁 A ≡          A+ ⋯
```

### Tooltip

`A` показывает связи.

`≡` показывает группы.

### `A+`

Поведение зависит от того, совпадает ли активный ASIN с существующей связью.

## 9.9. Состояние `linked + rejected`

### Условия

* товар связан с ASIN;
* товар отклонён.

### Иконки

```text
👁 A ×          A+ ⋯
```

### Требование

Это состояние разрешено, но должно быть визуально заметно как потенциально странное.

Tooltip `×` должен указывать, что связи не удалены:

```text
Отклонён
Связи с ASIN сохранены
```

## 9.10. Состояние `conflict`

### Условия

* пользователь пытается привязать товар к ASIN;
* у товара уже есть связь с другим ASIN;
* действие может заменить или добавить связь.

### Иконки

```text
👁 A!           A+ ⋯
```

### Главное правило

Нельзя автоматически заменять существующую связь без явного подтверждения пользователя.

# 11. Toast-уведомления и Undo

## 11.1. Назначение

Toast нужен для быстрой обратной связи после действий.

Каждое действие, меняющее данные, должно показывать toast.

## 11.2. Расположение

Рекомендуемое расположение: нижний правый угол или рядом с плавающей панелью, поверх страницы.

Toast не должен сдвигать страницу.

## 11.3. Undo

Для действий, которые меняют состояние, нужен `Undo`, если это возможно.

Действия с Undo:

* привязка к ASIN;
* добавление в группу;
* отклонение;
* отложение;
* снятие статуса;
* удаление связи;
* массовые действия.

## 11.4. Примеры toast

### Привязка

```text
Товар привязан к B0C7G00SE12
[Отменить]
```

### Добавление в группу

```text
Товар добавлен в группу «Проверить позже»
[Отменить]
```

### Отклонение

```text
Товар отклонён
[Отменить]
```

### Отложение

```text
Товар отложен для проверки
[Отменить]
```

### Ошибка

```text
Не удалось выполнить действие
Попробуйте ещё раз
```

## 11.5. Срок жизни toast

Обычный toast может исчезать через 3–5 секунд.

Toast с Undo должен оставаться достаточно долго, чтобы пользователь успел среагировать. Рекомендуется 5–8 секунд.

# 12. Конфликты и правила безопасности

## 12.1. Типы конфликтов

### Мягкий конфликт

Пример:

* товар уже в группе, пользователь добавляет его к ASIN;
* товар уже seen, пользователь добавляет его в группу.

Действие разрешается без предупреждения.

### Средний конфликт

Пример:

* товар уже связан с тем же ASIN;
* товар уже находится в той же группе.

Дубликаты не создаются. Показывается краткий toast.

### Жёсткий конфликт

Пример:

* товар уже связан с другим ASIN;
* товар rejected, но пользователь пытается привязать его к ASIN;
* массовая операция затрагивает товары с разными существующими связями.

Требуется явный выбор пользователя.

## 12.2. Главное правило конфликтов

Нельзя молча выполнять потенциально опасные действия:

* заменять существующий ASIN;
* удалять связь;
* удалять товар из группы;
* отклонять товар с удалением связей;
* массово перезаписывать данные.

## 12.3. Стандартный conflict-popover

```text
Найден конфликт

Что уже есть:
{current_context}

Новое действие:
{new_action_context}

[Оставить как есть]
[Добавить новое]
[Заменить]
[Отмена]
```

Кнопки должны называться конкретно для контекста. Например:

```text
[Добавить вторую связь]
[Заменить связь]
```

а не абстрактно:

```text
[Да]
[Нет]
```

# 14. Статистика страницы

## 14.1. Назначение

Статистика даёт быстрый обзор текущей WB-выдачи.

## 14.2. Где показывается

В плавающей панели.

## 14.3. Что считать

Минимально:

* общее количество карточек на странице;
* количество уникальных WB SKU;
* количество новых товаров;
* количество seen;
* количество товаров с ASIN-связями;
* количество товаров, связанных с активным ASIN;
* количество товаров, связанных с другими ASIN;
* количество товаров в группах;
* количество rejected;
* количество deferred;
* количество выбранных чекбоксами.

## 14.4. Обновление статистики

Статистика должна обновляться при:

* загрузке страницы;
* скролле и появлении новых карточек;
* изменении выдачи WB;
* выборе активного ASIN;
* любом действии пользователя;
* массовом действии;
* снятии выбора.

# 15. Поиск ASIN

## 15.1. Где используется

Поиск ASIN используется в двух местах:

1. в плавающей панели для выбора активного ASIN;
2. в меню карточки или массовых действиях для выбора конкретного ASIN.

## 15.2. Принцип поиска

Пользователь вводит любые данные. Система ищет по всем доступным полям.

Пользователь не должен выбирать фильтр вручную.

## 15.3. Поля поиска

* ASIN;
* title;
* brand;
* comment;
* category;
* keywords;
* workflow status;
* дата добавления;
* любые дополнительные полезные поля таблицы.

## 15.4. Ранжирование результатов

Рекомендуемый порядок приоритета:

1. точное совпадение ASIN;
2. ASIN начинается с введённой строки;
3. точное совпадение по бренду;
4. совпадение в названии;
5. совпадение в комментарии;
6. совпадение в ключевых словах;
7. недавно использованные;
8. активные за последние 30 дней;
9. в статусе `in_progress`.

## 15.5. Визуальный результат

Результаты должны быть плитками, а не голым списком.

```text
[image]
ASIN
Title
Brand
Comment
Status
[Select/Add]
```

# 16. Поиск групп

## 16.1. Где используется

* меню карточки `Добавить в группу...`;
* массовое действие `В группу...`;
* popover управления группами по клику на `≡`.

## 16.2. Поля поиска

* название группы;
* комментарий;
* иконка/тип;
* частота использования;
* дата последнего использования.

## 16.3. Пустой поиск

Если строка пустая, показывать:

1. часто используемые группы за последние 30 дней;
2. недавно использованные группы;
3. предустановленные рабочие группы;
4. самые крупные группы.

## 16.4. Создание группы

Форма:

```text
Название группы *
Иконка
Комментарий
```

# 18. Основные пользовательские сценарии

## 18.1. Сценарий A: быстрый матч активного ASIN

1. Пользователь открывает WB-страницу.
2. Пользователь открывает панель `WB ↔ A`.
3. Пользователь выбирает активный ASIN.
4. Пользователь листает WB-выдачу.
5. На подходящей карточке нажимает `A+`.
6. Система создаёт связь WB ↔ active ASIN.
7. На карточке появляется `A`.
8. Показывается toast с Undo.

Минимальный путь после выбора ASIN: один клик на карточке.

## 18.2. Сценарий B: необычная рекомендация WB, привязка к другому ASIN

1. Активный ASIN выбран.
2. Пользователь видит WB-товар, который подходит не к активному ASIN, а к другому Amazon-товару.
3. Пользователь нажимает `⋯` на карточке.
4. Выбирает `Добавить к ASIN...`.
5. Открывается поиск ASIN.
6. Если строка пустая, показываются недавние/активные ASIN.
7. Пользователь вводит часть ASIN, названия, бренда или комментария.
8. Система показывает кандидатов с фото, ASIN и названием.
9. Пользователь выбирает нужный ASIN.
10. Система создаёт связь WB ↔ выбранный ASIN.
11. Карточка обновляет `A`.
12. Показывается toast.

## 18.3. Сценарий C: добавить товар в группу

1. Пользователь нажимает `⋯`.
2. Выбирает `Добавить в группу...`.
3. Открывается поиск групп.
4. Если строка пустая, показываются частые группы.
5. Пользователь выбирает группу или создаёт новую.
6. Система добавляет WB-товар в группу.
7. На карточке появляется `≡`.
8. Показывается toast.

## 18.4. Сценарий D: отклонить товар

1. Пользователь нажимает `⋯`.
2. Выбирает `Отклонить`.
3. Выбирает или вводит причину.
4. Подтверждает.
5. На карточке появляется `×`.
6. Показывается toast с Undo.

## 18.5. Сценарий E: отложить товар

1. Пользователь нажимает `⋯`.
2. Выбирает `Отложить / проверить позже`.
3. Выбирает или вводит причину.
4. Подтверждает.
5. На карточке появляется `?` или `⏳`.
6. Показывается toast с Undo.

## 18.6. Сценарий F: товар уже связан с другим ASIN

1. Пользователь нажимает `A+`.
2. Система видит, что товар уже связан с другим ASIN.
3. Открывается conflict-popover.
4. Пользователь выбирает:

   * оставить как есть;
   * добавить вторую связь;
   * заменить связь;
   * отменить.
5. Система выполняет выбранное действие.
6. Показывается toast.

## 18.7. Сценарий G: массовая привязка к выбранному ASIN

1. Пользователь выделяет несколько карточек.
2. В панели появляется блок массовых действий.
3. Пользователь выбирает `К выбранному ASIN...`.
4. Открывается поиск ASIN.
5. Пользователь выбирает ASIN.
6. Система анализирует конфликты.
7. Если конфликтов нет, предлагает подтвердить массовую привязку.
8. Если конфликты есть, показывает варианты обработки.
9. После подтверждения выполняет действие.
10. Карточки обновляются.
11. Показывается toast.

# 19. Правила видимости и приоритетов иконок

## 19.1. Всегда видимые элементы

Для релевантной карточки всегда должны быть доступны:

* `A+`;
* `⋯`.

Исключение: если карточка не распознана как товар WB или нет SKU, элементы могут не отображаться.

## 19.2. Условно видимые элементы

* `○` — только если нет другого контекста.
* `👁` — если товар seen.
* `A` — если есть ASIN-связь.
* `≡` — если есть группы.
* `×` — если rejected.
* `?` или `⏳` — если deferred.

## 19.3. Если слишком много статусов

При ограниченной ширине карточки приоритет видимости:

1. `A` или `A!`;
2. `×`;
3. `?` / `⏳`;
4. `≡`;
5. `👁`;
6. `○`.

`A+` и `⋯` не скрывать.

Если статусов слишком много, допускается компактный агрегатор:

```text
+3
```

Но tooltip агрегатора должен показать все скрытые статусы.

# 20. Ошибки и пустые состояния

## 20.1. Нет активного ASIN

При нажатии `A+`:

```text
Активный ASIN не выбран

[Выбрать ASIN]
[Добавить к ASIN через поиск]
```

## 20.2. Поиск ASIN ничего не нашёл

```text
Ничего не найдено

Попробуйте ASIN, название, бренд или комментарий
```

Если возможно:

```text
[Показать недавние ASIN]
```

## 20.3. Поиск групп ничего не нашёл

```text
Группы не найдены

[Создать новую группу]
```

## 20.4. Не удалось сохранить действие

```text
Не удалось сохранить действие
Проверьте соединение и попробуйте ещё раз
```

Если действие было показано в UI оптимистично, нужно либо откатить UI, либо явно показать, что сохранение не завершено.

## 20.5. SKU не найден

Если расширение не может определить SKU карточки:

* не показывать `A+`;
* можно показать disabled `⋯` или не показывать элементы;
* при попытке действия показать:

```text
Не удалось определить WB SKU
Действие недоступно для этой карточки
```

# 21. Нефункциональные UX-требования

## 21.1. Скорость

Действия на карточке должны ощущаться мгновенными.

Пользователь не должен ждать заметной задержки после клика `A+`, открытия меню или hover-tooltip.

## 21.2. Работа с сотнями карточек

Интерфейс должен быть рассчитан на страницы с большим количеством карточек.

Поведение должно оставаться стабильным при:

* ручном скролле;
* динамической подгрузке товаров;
* обновлении выдачи;
* массовых действиях;
* открытии и закрытии панели.

## 21.3. Не ломать WB

Элементы расширения не должны:

* ломать клики по штатным ссылкам WB;
* мешать открытию карточки товара;
* перекрывать кнопку избранного WB без необходимости;
* менять штатные размеры карточек;
* создавать горизонтальный скролл;
* вызывать скачки layout.

## 21.4. Визуальная компактность

Все элементы расширения должны выглядеть как лёгкий слой поверх WB, а не как отдельная админка.

Основной интерфейс должен быть виден только тогда, когда он нужен.

Где:

* `⋯` — дополнительные действия.

Не показывать текстовые плашки на фото. Вся информация должна передаваться через иконки, tooltips, popovers и панель.

# 26. Критерии готовности UX

Интерфейс считается соответствующим ТЗ, если пользователь может:

1. открыть WB-страницу и увидеть компактные кнопки на карточках;
2. выбрать активный ASIN в плавающей панели;
3. одним кликом `A+` связать WB-товар с активным ASIN;
4. через `⋯` связать WB-товар с любым другим ASIN;
5. через `⋯` добавить WB-товар в группу;
6. увидеть, что товар уже seen;
7. увидеть, что товар уже связан с ASIN;
8. увидеть, что товар уже в группе;
9. увидеть, что товар rejected или deferred;
10. получить tooltip с контекстом по каждой статусной иконке;
11. получить предупреждение при конфликте ASIN;
12. выполнить массовое действие с выбранными товарами;
13. отменить последнее действие через toast;
14. не видеть поломки сетки WB;
15. не видеть крупных текстовых плашек поверх фото;
16. работать без постоянной боковой панели, с временной overlay-панелью;
17. понять текущий статус товара без открытия отдельной админки.

# 27. Финальное резюме логики

Расширение должно быть спроектировано вокруг трёх быстрых вопросов на каждой WB-карточке:

```text
1. Я уже взаимодействовал с этим товаром?
2. Он уже связан с ASIN, группой или решением?
3. Что я могу сделать с ним прямо сейчас одним кликом?
```

Ответы должны даваться через компактные иконки и tooltips.

Главное действие — `A+`, то есть привязка к активному ASIN.

Гибкие действия — через `⋯`, включая выбор любого ASIN и любой группы.

Плавающая панель нужна только для выбора активного ASIN, режима работы, статистики и массовых действий. Она не должна превращаться в постоянную рабочую область, которая меняет layout Wildberries.

Главный UX-результат:

> Пользователь должен быстрее, чем вручную с копированием ссылок, сопоставлять перспективные Amazon ASIN с товарами Wildberries, оставаясь внутри потока просмотра WB-выдачи.
